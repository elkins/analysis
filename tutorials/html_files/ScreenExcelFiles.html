Excel Files for AnalysisScreen Import

The advantage of importing your screening data with an Excel lookup file is not only that you can import hundreds or thousands of Spectra and Substances (compounds) in one go, but that the program will additionally automatically link the Substances to their Reference Spectra and place the Spectra into SpectrumGroups. And by including information such as compound SMILES you will also be able to see a compound's structure.

The program can read .xls or .xlsx files with multiple sheets that include the words Sample or Substance in the sheet name. You can create files that contain either the Substance or Sample page or both. You can download both a template Excel lookup file (only contains the column headings) and an example Excel lookup file (also contains example data).

Substance Sheet

The title of this sheet must start with Substance and contains information about the small molecules or compounds in your screening library. The only mandatory column is the one called substanceName. Additional columns are optional, but it can be helpful to use them, particularly if you want to include reference spectra for your small molecules.

spectrumPath

This is the path to a reference spectrum for that substance.
AnalysisScreen will recognise most spectrum formats, and you do not need to include any file extensions such as .ucsf, ndf5, .ft etc. You can specify the path in three ways:

  1. If all the spectra files are located in the same directory as the lookup file, insert only the file names.
  2. If the spectra are located in a subdirectory, insert the directory name first followed by a slash and the filename (the relative path starting from the Excel file), e.g. references/C101
  3. If the spectra files are located in a completely different location, insert the full path, e.g. /Users/Username/Desktop/MySpectra/C101
For Bruker files, you can insert the path to the 1r file, e.g.: /Users/Username/MySpectra/exptNumber/pdata/1/1r

spectrumGroupName

You can place the reference spectra for your substances into SpectrumGroups. Typically, they would all go into a SpectrumGroup called References, but you can give it a different name or put different Substances into different SpectrumGroups if you want to.

experimentType

Here you can enter the experiment Type such as 19F, 1H, STD or Water-LOGSY. Note that this is simply for users to keep track of experiment types in the Excel file. This information is not currently used by the program.

spectrumHexColour

Here you can specify the colour for the Reference Spectrum in hexadecimal notation. This can be useful if you want all References Spectra to be the same colour, for example. Enter is as, e.g. #BA55D3.

smiles

Entering the SMILES string for your Substance will enable you to view the structure of the compound within the program and also in the reports you generate.

other

There are many other Substance properties that you can add to the Excel file which will then be read into and stored in your project. These won't currently be used by the program, but will be available to you for reference.

Sample Sheet

The title of this sheet must start with Sample and contains information about the samples used in your screen. The only mandatory column is the one called sampleName. Additional columns are optional, but it can be helpful to use them, particularly if you want to create SpectrumGroups automatically, colour your spectra in particular ways or create links to the reference spectra. You can either place all your samples into one sheet or (what we recommend for clarity) have different sheets e.g. for your Control, Target and Displacer spectra.

spectrumGroupName

Enter the name of the spectrumGroup you would like your sample's spectrum to be included in.

spectrumPath

This is the path to the spectrum for a particular sample.
AnalysisScreen will recognise most spectrum formats, and you do not need to include any file extensions such as .ucsf, ndf5, .ft etc. You can specify the path in three ways:

  1. If all the spectra files are located in the same directory as the lookup file, insert only the file names.
  2. If the spectra are located in a subdirectory, insert the directory name first followed by a slash and the filename (the relative path starting from the Excel file), e.g. references/control_mix1
  3. If the spectra files are located in a completely different location, insert the full path, e.g. /Users/Username/Desktop/MySpectra/control_mix1
For Bruker files, you can insert the path to the 1r file, e.g.: /Users/Username/MySpectra/exptNumber/pdata/1/1r

spectrumName

Enter a SpectrumName if you don't want to use the program's default spectrum names (which are not very human friendly for Bruker data, for example).

experimentType

Here you can enter the experiment Type such as 19F, 1H, STD or Water-LOGSY. Note that this is simply for users to keep track of experiment types in the Excel file. This information is not currently used by the program.

spectrumHexColour

Here you can specify the colour for the sample Spectrum in hexadecimal notation. This can be particularly useful if you want all control or target spectra to be the same colour, for example. Enter is as, e.g. #1E90FF.

sampleComponents

Insert the names of the compounds (Substances) that are present in the sample. In the case of a mixture containing components 1 to 5, insert them as a comma-separated list without spaces, e.g. C101,C102,C103,C104,C105.
